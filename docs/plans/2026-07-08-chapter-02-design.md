## 章节定位

- **目标读者**：AI、材料科学等理工科初学者，平时习惯图形界面，命令行基础薄弱，需要从零建立对命令行的信心。
- **与前后章衔接**：
  - **承前**：第一章已完成 Linux 系统安装、登录与桌面环境的基本认知，本章进入真正的命令行世界。
  - **启后**：第三章将展开用户、权限与软件安装，本章为其奠定“为什么用、怎么用、怎么求助”的基础；其中远程连接、文件系统导航、环境变量等概念将直接延续到第三章。
- **教学目标**：
  1. 理解命令行相比 GUI 的价值与不可替代场景。
  2. 区分终端、Shell、Bash 三个概念，看懂提示符，了解 Bash 配置文件与别名。
  3. 能通过 SSH 远程登录服务器，并理解密码认证与密钥认证的区别。
  4. 掌握 Linux 文件系统的根目录、路径与基本导航方法。
  5. 能创建、复制、移动、删除文件和目录，能使用通配符批量操作。
  6. 理解管道与重定向，能组合简单命令完成批量任务。
  7. 养成使用帮助、谨慎操作、先想再执行的好习惯。

### 1. 为什么还需要命令行？

#### 1.1 批量任务：一行命令胜过一百次点击

- **内容要点**：
  - 对比 GUI 的逐次点击与 CLI 的精确表达。
  - 强调命令行的优势在于可记录、可复现、可自动化。
  - 不贬低 GUI：GUI 适合探索，CLI 适合批量与远程。
- **涉及命令/概念**：CLI、GUI、脚本（仅作概念介绍，不写脚本语法）。
- **示例场景**：从 100 个 `result_*.log` 文件中提取所有 `Final Accuracy`。
  - 示例命令（仅示意，不提前讲解全部语法）：
    ```bash
    grep "Final Accuracy" result_*.log > accuracy_summary.txt
    ```

### 2. Shell、终端、Bash 与配置文件

#### 2.1 终端是窗户，Shell 是翻译

- **内容要点**：
  - 终端（Terminal）是负责显示文字、接收键盘输入的程序窗口，本身不执行命令。
  - Shell 是解释你输入的命令、并把它转交给内核执行的程序。
  - 内核执行完后再把结果交给 Shell，Shell 通过终端把输出显示给你。
  - 类比：终端是窗户，Shell 是站在窗边帮你翻译和传话的人，内核是屋里真正干活的。
- **涉及命令/概念**：终端（Terminal）、Shell、内核（Kernel）、标准输入、标准输出。
- **示例场景**：
  - 打开“终端”应用，输入 `ls`，按下回车。
  - 流程：终端把 `ls` 传给 Shell → Shell 解析后调用 `/bin/ls` → 内核执行 → 结果返回 Shell → 终端显示文件列表。
- **配图建议**：图 2.4 `fig-2.4-terminal-shell-kernel.svg`（已生成，原图 2.2 重编号）。
- **与前后文衔接**：
  - 承前：回答第一节“为什么用命令行”中提到的“敲命令”到底是谁在接收、谁在执行。
  - 启后：为 2.5 节“输入命令后发生了什么”和第六节管道与重定向埋下输入/输出概念。

#### 2.2 Bash 只是众多 Shell 中的一种

- **内容要点**：
  - Bash 是目前 Linux 发行版中最常见的默认 Shell，但 Shell 不只有 Bash。
  - 简要提及 sh、zsh、fish 等存在即可，不比较优劣，不讲解切换与配置。
  - 本书所有示例默认使用 Bash，学生无需额外设置。
- **涉及命令/概念**：Bash、sh、zsh、fish、默认 Shell。
- **示例场景**：
  ```bash
  echo $SHELL
  # 输出：/bin/bash
  bash --version
  ```
- **配图建议**：可仅用文字说明，无需新增配图；如需要，可用简单文字列表展示常见 Shell。
- **与前后文衔接**：

#### 2.3 看懂提示符：它其实在向你汇报

- **内容要点**：
  - Bash 默认提示符通常形如 `用户名@主机名:当前目录$`。
  - `$` 表示当前是普通用户；`#` 表示当前是 root（超级用户），要格外小心。
  - 提示符中的当前目录帮助你判断“我现在在哪里”，避免误操作。
  - 不要看到 `#` 就随意执行命令，root 权限可以删除系统文件。
- **涉及命令/概念**：提示符（prompt）、当前目录、用户身份、`$`、`#`、root。
- **示例场景**：
  ```bash
  student@gpu-lab01:~/Documents$
  ```
  逐段解释：`student` 用户名、`gpu-lab01` 主机名、`~/Documents` 当前目录、`$` 普通用户。

#### 2.4 输入命令后发生了什么

- **内容要点**：
  - 简要流程：Shell 读取输入 → 拆分为命令名、选项、参数 → 在 PATH 中查找对应程序 → 调用程序执行 → 程序把结果写到标准输出 → Shell 把结果显示到终端。
  - 目的不是讲操作系统原理，而是让学生知道“命令不是魔法”，减少畏惧感。
  - 标准输入/输出在此轻轻带过，为第六节“管道与重定向”留伏笔。
- **涉及命令/概念**：命令解析、选项（flag）、参数、PATH（概念提及）、程序调用、标准输入/标准输出。
- **示例场景**：
  ```bash
  ls -l /home
  ```
  拆解：`ls` 是命令，`-l` 是选项，`/home` 是参数；Shell 找到 `/bin/ls` 并执行，输出目录详情。

#### 2.5 Bash 配置文件：`.bashrc`、`.bash_profile`、`.profile`

- **内容要点**：
  - Bash 启动时会读取一些配置文件，用来设置别名、环境变量、提示符样式等。
  - `.bashrc`：每次打开新的交互式 Shell 时读取，适合放别名和函数。
  - `.bash_profile` 或 `.profile`：登录时读取，适合放只在登录时需要一次的环境变量。
  - 初学者只需知道它们存在、不要乱改；修改后可用 `source` 命令立即生效。
  - 不要混淆“登录 Shell”与“非登录 Shell”的细节，点到为止。
- **涉及命令/概念**：`.bashrc`、`.bash_profile`、`.profile`、`source`、登录 Shell、非登录 Shell（概念提及）。
- **示例场景**：
  ```bash
  ls -la ~ | grep bash
  # 查看已有的 Bash 配置文件
  source ~/.bashrc
  # 修改后让配置立即生效
  ```

#### 2.6 别名：`alias`

- **内容要点**：
  - 别名是给长命令起短名字，减少重复输入。
  - 常用示例：`alias ll='ls -l'`、`alias ..='cd ..'`。
  - 别名只在当前 Shell 有效；要永久保存，需写入 `.bashrc`。
  - 强调别名适合个人习惯，不适合替代对原命令的理解。
- **涉及命令/概念**：`alias`、`unalias`、`.bashrc`、命令缩写。
- **示例场景**：
  ```bash
  alias ll='ls -lha'
  ll
  # 等效于 ls -lha
  alias today='date +%Y-%m-%d'
  today
  # 输出当前日期
  ```

### 3. 远程连接：SSH 与密钥

#### 3.1 为什么需要 SSH

- **内容要点**：
  - SSH（Secure Shell）是加密的远程登录协议，用于在不安全的网络中安全地操作远程机器。
  - 实验室服务器、云主机、HPC 集群几乎全靠 SSH 管理。
  - SSH 不只是登录，还能传文件、执行远程命令，但本章聚焦登录。
- **涉及命令/概念**：SSH、远程登录、加密、服务器、客户端。
- **示例场景**：
  ```bash
  ssh student@gpu-lab01
  ```
- **配图建议**：图 2.8 SVG 示意图，展示本地电脑与远程服务器之间建立加密通道。
- **与前后文衔接**：
  - 承前：第一节已经提到远程服务器是命令行的主战场。
  - 启后：3.2 节讲解第一次登录时的安全确认。

#### 3.2 第一次登录：host key 与指纹

- **内容要点**：
  - 第一次连接某台服务器时，SSH 会提示保存服务器的 host key 指纹，用于防止中间人攻击。
  - 如果指纹变了，SSH 会发出警告，这时要联系管理员确认，不要盲目继续。
  - 已保存的 host key 存放在 `~/.ssh/known_hosts`。
- **涉及命令/概念**：host key、指纹（fingerprint）、`known_hosts`、中间人攻击（概念提及）。
- **示例场景**：
  ```bash
  ssh student@gpu-lab01
  # 提示：The authenticity of host 'gpu-lab01' can't be established.
  # 显示 ECDSA key fingerprint 并询问是否继续
  ```
- **配图建议**：图 2.9 SVG 终端窗口图，展示第一次 SSH 登录时的提示信息及逐段解释。
- **与前后文衔接**：
  - 承前：3.1 说明 SSH 是什么，本节进入第一次登录的真实体验。
  - 启后：3.3 节比较密码认证与密钥认证。

#### 3.3 密码认证 vs 密钥认证

- **内容要点**：
  - 密码认证：每次登录输入密码，简单但容易受暴力破解，也不适合自动化。
  - 密钥认证：本地保存私钥，服务器保存公钥，登录时自动验证，更安全、更方便。
  - `ssh-keygen` 生成密钥对；`ssh-copy-id` 把公钥复制到服务器。
  - 私钥文件（如 `id_rsa`、`id_ed25519`）必须妥善保管，不能泄露。
- **涉及命令/概念**：`ssh-keygen`、`ssh-copy-id`、公钥、私钥、`~/.ssh/id_*`、`~/.ssh/authorized_keys`。
- **示例场景**：
  ```bash
  ssh-keygen -t ed25519 -C "student@example.com"
  ssh-copy-id student@gpu-lab01
  ssh student@gpu-lab01
  # 无需输入密码即可登录
  ```
- **配图建议**：图 2.10 SVG 示意图，展示公钥/私钥配对验证过程。
- **与前后文衔接**：
  - 承前：3.2 完成第一次登录，本节升级为更安全的登录方式。
  - 启后：3.4 节把常用 SSH 选项写进配置文件，进一步简化登录。

#### 3.4 `~/.ssh/config` 与常用技巧

- **内容要点**：
  - 把服务器地址、用户名、端口、私钥路径写进 `~/.ssh/config`，可用简短别名登录。
  - 常用配置字段：`Host`、`Hostname`、`User`、`Port`、`IdentityFile`。
  - 简单技巧：`ssh lab` 替代 `ssh -p 2222 student@gpu-lab01.example.com`。
  - 强调 `~/.ssh/config` 和私钥文件权限要严格控制（简要提及，不深究权限数字）。
- **涉及命令/概念**：`~/.ssh/config`、`Host`、`Hostname`、`User`、`Port`、`IdentityFile`。
- **示例场景**：
  ```bash
  # ~/.ssh/config 示例
  Host lab
      Hostname gpu-lab01.example.com
      User student
      Port 2222
      IdentityFile ~/.ssh/id_ed25519
  ```
  然后执行：
  ```bash
  ssh lab
  ```
- **配图建议**：图 2.11 SVG 示意图，对比“长命令”与“简短别名”两种登录方式。
- **与前后文衔接**：
  - 承前：3.3 配置好密钥后，本节进一步把登录信息固化成别名。
  - 启后：第四节开始讲登录后在服务器上看到的文件系统。

### 4. 文件系统：从 `/` 开始的世界

#### 4.1 没有 C 盘，一切从 `/` 开始

- **内容要点**：
  - Linux 没有 Windows 式的 C 盘、D 盘，所有文件和目录都从唯一的根目录 `/` 开始，形成一棵树。
  - 简要介绍顶层目录含义：`/home` 存放用户家目录，`/root` 是 root 用户家目录，`/tmp` 临时文件，`/etc` 配置文件，`/bin`/`/usr/bin` 常用程序，`/var` 日志等。点到为止，不展开。
  - 可顺带对比 Windows 盘符与 macOS 的 `/Users`，帮助学生迁移认知。
- **涉及命令/概念**：根目录（`/`）、目录树、文件系统层级标准（FHS，仅概念提及）、`/home`、`/root`、`/tmp`、`/etc`、`/bin`、`/usr`、`/var`、`/dev`。
- **示例场景**：
  ```bash
  ls /
  # 输出：bin  boot  dev  etc  home  lib  tmp  usr  var ...
  ```
- **配图建议**：图 2.12 `fig-2.12-linux-directory-tree.svg`（待新建：Linux 目录树示意，原图 2.5 重编号）。
- **与前后文衔接**：
  - 承前：第二节提示符里已经出现“当前目录”，本节解释目录从哪来。
  - 启后：从根目录自然过渡到家目录，再进入相对路径。

#### 4.2 家目录与波浪号

- **内容要点**：
  - 每个用户都有一个家目录，普通用户通常在 `/home/用户名`。
  - `~` 是家目录的简写，`~/Documents` 等价于 `/home/student/Documents`。
  - `cd` 不加参数或 `cd ~` 都会回到家目录。
- **涉及命令/概念**：家目录（home directory）、`~`、`$HOME`、`cd`。
- **示例场景**：
  ```bash
  pwd
  # /home/student
  cd ~/Documents
  pwd
  # /home/student/Documents
  cd
  pwd
  # /home/student
  ```
- **配图建议**：图 2.13 `fig-2.13-home-directory.svg`（待新建：家目录位置与 `~` 简写示意，原图 2.6 重编号）。
- **与前后文衔接**：
  - 承前：根目录下最重要的分支之一就是 `/home`。

#### 4.3 `.` 和 `..`：当前目录与上一级

- **内容要点**：
  - `.` 代表当前目录，`..` 代表当前目录的父目录。
  - 注意区分：以 `.` 开头的文件是隐藏文件，与 `.` 目录本身不同。
  - `.` 常用于执行当前目录下的脚本（如 `./script.sh`），`..` 用于返回上一级。
- **涉及命令/概念**：`.`、`..`、当前目录、父目录、隐藏文件。
- **示例场景**：
  ```bash
  cd /home/student/Documents
  cd ..
  pwd
  # /home/student
  ls .
  # 列出当前目录内容
  ```

#### 4.4 绝对路径与相对路径

- **内容要点**：
  - 绝对路径从 `/` 开始，不随当前目录变化，如 `/home/student/Documents/report.txt`。
  - 相对路径从当前目录开始，如 `Documents/report.txt` 或 `../backup`。
  - 两种路径没有绝对优劣：脚本和配置中常用绝对路径，日常操作常用相对路径。
  - 判断原则：看路径是否以 `/` 或 `~` 开头。
- **涉及命令/概念**：绝对路径、相对路径、路径起点。
- **示例场景**：
  - 当前目录为 `/home/student`：
    ```bash
    ls Documents/data.csv                # 相对路径
    ls /home/student/Documents/data.csv  # 绝对路径
    ls ~/Documents/data.csv              # 以 ~ 开头，也视为绝对路径
    ```

- **与前后文衔接**：
  - 承前：`.`、`..`、`~` 都是相对路径的重要符号。
  - 启后：路径概念是 `pwd`、`ls`、`cd` 三板斧的操作前提。

#### 4.5 三板斧：`pwd`、`ls`、`cd`

- **内容要点**：
  - `pwd`（print working directory）显示当前绝对路径，解决“我在哪”。
  - `ls` 列出目录内容，常用选项 `-l`（长格式）、`-a`（含隐藏文件）、`-h`（人类可读大小）。
  - `cd` 切换目录，可配合 `~`、`.`、`..`、`-`（返回上次目录）使用。
  - 三个命令配合即可完成日常目录导航。

- **涉及命令/概念**：`pwd`、`ls`、`cd`、`ls -la`、`cd -`、Tab 补全。
- **示例场景**：

  ```bash
  pwd

  ls
  # Documents  Downloads  Music
  cd Documents
  ls -la

  cd -
  ```

### 5. 常用命令工具箱

#### 5.1 创建与删除：`touch`、`mkdir`、`rm`、`rmdir`

- **内容要点**：
  - `touch` 创建空文件或更新文件时间戳；初学者先掌握“创建空文件”即可。
  - `mkdir` 创建目录，`mkdir -p` 可一次性创建多级目录。
  - `rm` 删除文件，`rm -r` 删除目录及其内容；`rm` 不可恢复，必须谨慎。
  - `rmdir` 仅删除空目录，安全性高于 `rm -r`，适合初学者练习。
  - 强调删除前先用 `ls` 确认目标，避免误删。
- **涉及命令/概念**：`touch`、`mkdir`、`mkdir -p`、`rm`、`rm -i`、`rm -r`、`rmdir`。
- **示例场景**：
  ```bash
  touch notes.txt
  mkdir lab_reports
  mkdir -p backup/2026/July
  rm notes.txt
  rmdir lab_reports
  rm -r backup/2026/July   # 危险：会递归删除
  ```
- **配图建议**：图 2.16 `fig-2.16-create-delete.svg`（待新建：创建与删除命令示意，原图 2.9 重编号）。
- **与前后文衔接**：

  - 启后：复制与移动是创建之后的自然延伸。

#### 5.2 复制与移动：`cp` 与 `mv`

- **内容要点**：
  - `cp` 用于复制文件或目录，复制目录需加 `-r`。
  - `mv` 用于移动或重命名；在同一目录下 `mv 旧名 新名` 即重命名。
  - 目标路径已存在同名文件时，默认会覆盖，可用 `-i` 开启交互确认。
  - 复制/移动前先用 `ls` 查看源和目标，避免覆盖重要文件。
- **涉及命令/概念**：`cp`、`cp -r`、`mv`、`mv -i`、覆盖。
- **示例场景**：
  ```bash
  cp notes.txt notes_backup.txt
  cp -r lab_reports lab_reports_copy
  mv notes.txt diary.txt          # 重命名
  mv diary.txt archive/           # 移动
  mv -i diary.txt archive/        # 若目标存在则询问
  ```
- **配图建议**：图 2.17 `fig-2.17-copy-move.svg`（待新建：复制与移动示意，原图 2.10 重编号）。
- **与前后文衔接**：
  - 承前：5.1 学会创建与删除，本节学习“改名”和“复制备份”。
  - 启后：批量复制/移动自然引出通配符。

#### 5.3 隐藏文件与文件名规则

- **内容要点**：
  - 以 `.` 开头的文件/目录为隐藏文件，`ls` 默认不显示，需用 `ls -a`。
  - Linux 文件名区分大小写，`Report.txt` 与 `report.txt` 是两个文件。
  - 文件名中避免空格；若必须用，可用引号包裹或转义。
  - 文件名不能包含 `/`，避免使用 `*`、`?`、`$` 等特殊字符以减少麻烦。
  - 推荐命名习惯：小写字母、数字、下划线或连字符，如 `data_2026_07.csv`。
- **涉及命令/概念**：隐藏文件、`ls -a`、文件名大小写、命名规范。
- **示例场景**：
  ```bash
  touch .config
  ls
  # 不显示 .config
  ls -a
  # 显示 .config
  touch "my file.txt"   # 不推荐，但可行
  touch my_file.txt     # 推荐
  ```

#### 5.4 通配符：批量操作的秘密武器

- **内容要点**：
  - `*` 匹配任意数量字符（包括零个），如 `*.txt` 匹配所有 txt 文件。
  - `?` 匹配单个字符，如 `file?.txt` 匹配 `file1.txt` 但不匹配 `file10.txt`。
  - `[abc]` 匹配方括号内任一字符，如 `file[123].txt`。
  - 通配符由 Shell 展开，命令实际接收到的是匹配后的文件列表。
  - 配合 `cp`、`mv`、`rm` 可一次性处理多个文件，但批量删除前务必先用 `ls` 预览。
- **涉及命令/概念**：通配符、`*`、`?`、`[]`、Shell 展开。
- **示例场景**：
  ```bash
  ls *.txt
  cp *.txt backup/
  mv report?.pdf archive/
  rm temp[0-9].log
  ```

#### 5.5 `find`：在文件系统里找人

### 6. 管道与重定向：命令行的“乐高”

#### 6.1 管道：把命令串起来

- **内容要点**：
  - 管道符 `|` 将前一个命令的输出作为后一个命令的输入。
  - 管道让多个简单命令组合成复杂任务，体现“乐高式”组合思想。
  - 常用组合：`ls | wc -l`、`cat file | grep keyword`。
  - 每个命令只负责一件事，通过管道协作。
- **涉及命令/概念**：管道 `|`、命令组合、标准输出、标准输入。
- **示例场景**：
  ```bash
  ls | wc -l
  cat result.log | grep "Accuracy"
  grep "error" *.log | wc -l
  ```
- **配图建议**：图 2.22 `fig-2.22-pipe.svg`（待新建：管道连接示意，原图 2.19 重编号）。
- **与前后文衔接**：
  - 承前：第二节已轻触标准输入/输出概念，第五节已用 `grep` 等命令。
  - 启后：管道常与重定向一起使用，完成更完整的任务。

#### 6.2 重定向：输出到文件

- **内容要点**：
  - `>` 将输出覆盖写入文件，`>>` 将输出追加到文件末尾。
  - 重定向会创建不存在的文件，或覆盖已存在文件，使用 `>` 前需确认目标。
  - 可通过重定向保存命令结果、生成报告。
  - 与管道不同：管道连接命令，重定向连接命令与文件。
- **涉及命令/概念**：`>`、`>>`、输出重定向、覆盖、追加。
- **示例场景**：
  ```bash
  echo "实验开始" > log.txt
  echo "步骤 1 完成" >> log.txt
  ls *.csv > file_list.txt
  ```

- **与前后文衔接**：
  - 承前：命令默认输出到终端，本节学习把输出存到文件。
  - 启后：引入标准错误后，可区分“正常输出”和“错误输出”。

#### 6.3 `stdin`、`stdout`、`stderr`：三种水流

- **内容要点**：
  - `stdout`（标准输出）：命令正常结果，默认显示在终端。
  - `stderr`（标准错误）：命令报错信息，默认也显示在终端，但与 `stdout` 是两条通道。
  - `stdin`（标准输入）：命令读取数据的入口，默认是键盘。
  - `2>` 将错误信息重定向到文件，`2>&1` 仅作概念提及，不深入。
  - 本节目标是让学生知道“输出不是一股水”，为排查错误留基础。
- **涉及命令/概念**：`stdin`、`stdout`、`stderr`、`<`、`2>`、文件描述符（仅概念）。
- **示例场景**：
  ```bash
  ls existing.txt missing.txt > out.log 2> err.log
  cat < file.txt                         # 从文件读取输入
  ```

### 7. 迷路了怎么办？求助与好习惯

#### 7.1 先别急着敲回车

- **内容要点**：

  - 对不确定的命令，先用 `echo` 或 `ls` 预览效果，例如 `echo rm *.txt` 先看会删哪些。
  - 养成“停顿一秒”的习惯，可显著减少误操作。
- **涉及命令/概念**：`pwd`、`ls`、命令预览、`echo`。
- **示例场景**：
  ```bash
  pwd
  ls *.txt
  echo rm *.txt   # 预览，不会真正删除
  ```

#### 7.2 三条求助渠道：`man`、`--help`、`tldr`

  - `tldr 命令名` 提供常见用法示例（需安装），适合新手。
  - 推荐顺序：先 `--help` 或 `tldr` 快速上手，需要细节再查 `man`。
  - 不要害怕英文帮助，关键是找到“用法格式”和“常用选项”。
- **涉及命令/概念**：`man`、`--help`、`tldr`、`whatis`（可选提及）。
- **示例场景**：

  ```bash
  ls --help
  ```

#### 7.3 先 `ls` 再操作：肌肉记忆

- **内容要点**：
  - 执行 `rm`、`cp`、`mv` 等修改类命令前，先用 `ls` 确认目标存在且范围正确。
  - 使用通配符时，先用 `ls 通配符模式` 预览匹配结果。
  - 对批量删除等危险操作，可先用 `echo` 模拟。
  - 把“定位→预览→执行→验证”作为固定流程。
- **涉及命令/概念**：`ls`、预览、通配符、安全流程。
- **示例场景**：
  ```bash
  ls backup/
  rm backup/*.tmp          # 先 ls backup 确认过再执行
  ls *.log
  echo rm *.log            # 模拟，确认无误后再执行真正的 rm
  ```

- **与前后文衔接**：

  - 启后：7.4 介绍执行中/输入时的救场快捷键。

#### 7.4 `Ctrl+C` 与 `Tab`：两个救场键

- **内容要点**：
  - `Ctrl+C` 中断当前正在运行的命令，适用于死循环、长时间任务或误操作。
  - `Tab` 补全命令名和路径，减少拼写错误；按两次 `Tab` 列出候选。
  - `Ctrl+L` 清屏（可选提及），`Ctrl+D` 表示输入结束（可选提及，谨慎介绍）。
  - 强调 `Ctrl+C` 是“停止命令”，不是“复制”；初学者容易混淆。
- **涉及命令/概念**：`Ctrl+C`、`Tab`、`Ctrl+L`、`Ctrl+D`。
- **示例场景**：
  ```bash
  # 误启动了一个长时间命令，按 Ctrl+C 停止
  python train.py   # 按 Ctrl+C 可中断
  # 输入 cd /usr/loc 后按 Tab 补全为 cd /usr/local
  ```
- **配图建议**：可用 CSS 样式化代码块配合文字说明，无需新增配图。
- **与前后文衔接**：
  - 承前：7.3 讲执行前确认，本节讲执行中/输入时的补救。
  - 启后：7.5 汇总成安全清单。

#### 7.5 初学者的安全清单

- **内容要点**：
  - 汇总本章安全习惯：回车前停顿、先 `ls` 再操作、危险命令用 `-i`、不复制粘贴不明命令。
  - 永远不要随意执行 `rm -rf /` 或来历不明的脚本。
  - 重要数据操作前先备份；不确定时用 `echo` 或临时目录做测试。
  - 出错信息不是敌人，认真阅读错误提示往往能自己解决。
  - 给出一个可打印/可保存的简短清单。
- **涉及命令/概念**：安全操作、备份、`rm -rf`、错误信息阅读。
- **示例场景**：
  ```bash
  # 安全清单示例动作
  pwd && ls          # 先确认位置
  cp -i important.txt important.txt.bak   # 备份
  rm -i old.txt      # 交互删除
  ```

## 风险与注意事项

- **避免写成命令手册**：每个命令都要配合“为什么学”“什么时候用”的场景，避免堆砌参数列表。
- **示例安全**：所有命令示例使用虚构用户名/主机名（如 `student@gpu-lab01`），避免泄露真实信息。
- **危险命令警示**：涉及 `rm`、`>` 覆盖等操作时，必须在正文、注释或侧边提示中给出安全提醒。
- **SVG 与 CSS 代码块边界**：概念性、流程性、结构性的图用 SVG；具体命令与输出用 CSS 代码块，避免把长命令也做成图片（不利于复制与检索）。
- **读者心理建设**：反复强调“命令行不是背命令”，鼓励先理解意图、再查表达。
- **章节边界清晰**：本章不覆盖权限与软件安装，遇到相关需求时明确指向第三章，避免学生困惑。
