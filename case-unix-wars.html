Unix 战争与 BSD 诉讼案

Unix 曾经是学术界和黑客圈共同享有的财富，后来却变成了一场漫长的商业混战。理解这段历史，能帮我们明白为什么 Linux 会在 1991 年以那样一种方式出现——它不是偶然，而是 Unix 自己给自己制造了一个窗口期。

这里所说的“Unix 战争”，核心就是 AT&T 与 BSD 之间围绕 Unix 版权和商业控制权的长期争夺。

1. 从开放到封闭

Unix 诞生于贝尔实验室，但早期 AT&T 并没有把它当成主打产品。由于反垄断判决的限制，AT&T 不能进入计算机软件销售市场，于是 Unix 源码以几乎可以忽略的价格分发给大学。加州大学伯克利分校就是最大的受益者之一，BSD（Berkeley Software Distribution）就在那里逐渐成形。

转折发生在 1984 年。这一年，美国政府依据反垄断法拆分了 AT&T 的贝尔系统，禁令解除，AT&T 开始把 Unix 当作真正的商品来卖，商业版本价格昂贵，源码也不再像过去那样随意分发。Unix 从一个大家都能改、都能研究的系统，迅速变成了封闭的商业软件。

2. BSD 的崛起

AT&T 转向商业化的同时，伯克利的 BSD 研究团队却在另一个方向上越走越远。他们重写了大量内核代码，加入了 TCP/IP 网络协议栈，发展出独立的 BSD 许可证，并推出了 4.2BSD、4.3BSD 等版本。

到 20 世纪 80 年代末，BSD 已经是学术机构和互联网基础设施中非常重要的力量。很多人认为，BSD 有望成为 Unix 世界事实上的标准。但问题是，BSD 最初毕竟是从 AT&T 的 Unix 代码衍生出来的，版权归属一直是一笔糊涂账。

3. 诉讼爆发

1992 年，AT&T 的子公司 Unix System Laboratories（USL）对 BSDi 和加州大学伯克利分校提起诉讼，指控 BSD 侵犯了 Unix 的版权、泄露了商业秘密，并滥用了商标。诉讼的发起时间非常关键——它正好发生在个人电脑和互联网开始爆发的节点上。

这场诉讼持续了近两年。期间，BSD 的代码地位陷入法律不确定性，各大厂商和研究机构不敢再放心地基于 BSD 做新产品。原本有望成为主流自由 Unix 系统的 BSD，发展速度明显放慢。

4. 和解与余波

1994 年，双方达成和解。和解的条件之一是：BSD 必须删除约三处被认定来自 AT&T Unix 的核心文件，随后以 4.4BSD-Lite 的形式重新发布。这场诉讼在法律意义上结束了，但它已经深刻地改变了 Unix 世界的格局。

BSD 社区最终分化出 FreeBSD、OpenBSD、NetBSD 等多个分支，它们在服务器和网络设备领域一直保持影响力。但不可否认的是，诉讼让 BSD 错过了 1990 年代初期个人计算机和互联网基础设施高速扩张的最佳时机。

5. 为什么说这场诉讼给了 Linux 机会

1991 年，林纳斯·托瓦兹开始编写 Linux 时，他并不是在一个真空中工作。他需要的是一个自由的、类 Unix 的内核，而当时 BSD 正被官司缠住，Minix 又带有浓厚的教学限制。Linux 的出现正好填补了这块空白。

更重要的是，Linux 从一开始就没有任何 AT&T 代码，它在法律上是干净的。当 1994 年 BSD 诉讼和解时，Linux 已经通过 GPL 协议聚集了全球开发者，发展势头已经形成。历史没法假设，但如果没有这场诉讼拖慢 BSD，Linux 是否还能后来居上，恐怕会是一个完全不同的答案。
