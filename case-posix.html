POSIX 是什么？

一句话：POSIX 是一套“操作系统的通用接口规范”。它规定了操作系统应该提供哪些命令、文件路径、系统调用和编程接口，让程序在不同系统之间更容易移植。Linux 和 macOS 看起来很像，很大程度上就是因为它们都遵循了 POSIX。

全称与含义

POSIX 的全称是 Portable Operating System Interface，中文常译为“可移植操作系统接口”。名字里的 X 来自 UNIX，读作 /ˈpɒzɪks/，习惯上直接叫“POSIX”。

注意，POSIX 本身不是一个操作系统，也不是某个具体软件。它更像一本“约定手册”：如果你希望自己的系统长得像 Unix、能被 Unix 程序员熟悉，那就应该按照这本手册来实现命令和接口。

历史背景：Unix 分化后为什么需要统一接口

上世纪 70 年代，Unix 诞生后很快流行起来。但因为源代码授权给多家公司和大学，大家各自改造，逐渐分化出 System V、BSD、SunOS、AIX、HP-UX 等许多版本。

问题随之而来：同样一段 C 程序，在 A 家的 Unix 上能编译运行，到了 B 家的 Unix 上可能就报错；同一个 Shell 脚本，在这台服务器上正常，换到另一台就行为不同。每个系统都加了各自的“小扩展”，程序员苦不堪言。

为了结束这种混乱，IEEE 在 1988 年推出了 IEEE Std 1003 标准，后来被称为 POSIX。它的目标很简单：定义一套大家共同遵守的最小接口集合。只要系统符合 POSIX，软件就不需要为每个 Unix 变体单独重写。

主要内容：命令、文件系统、进程、线程与信号

POSIX 的内容很多，初学者不需要背下来，但要知道它大概覆盖了哪些方面：

  • Shell 与常用命令： 规定了 ls、cd、grep、awk、ps 等命令的基本行为，以及管道 |、重定向 >、环境变量、脚本语法等。
  • 文件系统路径： 规定以 / 为根目录，/bin、/etc、/home（或 /Users）等目录的含义，以及路径分隔符、文件权限模型。
  • 进程管理： 规定进程如何创建（fork）、替换（exec）、等待（wait）、退出并返回状态码，以及 ps、kill 等命令。
  • 线程： POSIX 线程（Pthreads）定义了跨系统的多线程编程接口，比如创建线程、加锁、条件变量等。
  • 信号： 规定进程间如何用信号通信，例如 SIGINT（Ctrl+C）、SIGTERM、SIGKILL 的含义和默认处理方式。
  • 系统调用与 C 语言接口： 规定打开文件（open）、读写（read/write）、关闭（close）、创建管道（pipe）等函数的原型和行为。

可以把 POSIX 理解为 Unix 世界的“普通话”：只要大家会说这种话，交流就不会太费劲。

与 Linux、macOS 和 Windows 的关系

POSIX 不是某个系统的专属标准，而是一把尺子，用来衡量一个系统“像不像 Unix”。

  • Linux： Linux 内核加上 GNU 工具链后，基本符合 POSIX。这也是 Linux 发行版能直接运行大量 Unix 软件的原因。
  • macOS： macOS 通过了 UNIX 认证，自然也满足 POSIX。它的底层是 BSD 一脉，所以有些命令的选项和 GNU/Linux 略有不同，但大体接口一致。
  • Windows： Windows 本身不是 POSIX 系统，它的原生 API 完全不同。但 Windows 提供了 WSL（Windows Subsystem for Linux）、Cygwin、MSYS2 等兼容层，让你能在 Windows 上运行 POSIX 风格的命令和程序。

所以，一个系统“符合 POSIX”并不意味着它跟 Linux 一模一样，而是说它提供了足够通用的接口，让熟悉 Linux/macOS 的人不会完全陌生。

实际意义：为什么同样的命令和脚本能通用

POSIX 最大的实际价值，就是“可移植性”。

想象一下：你写了一个 Shell 脚本，里面用 ls 列文件、用 grep 过滤、用 > 重定向输出、用 $? 判断上一条命令是否成功。只要这些用法都是 POSIX 标准支持的，你就可以把它从 Ubuntu 拷贝到 macOS，甚至拷贝到 FreeBSD，大概率都能跑。

同样，用 C 语言调用 open()、read()、fork() 的程序，也更容易在不同 Unix-like 系统上编译。

当然，POSIX 只规定了“共同底线”。很多系统还会提供自己的扩展，比如 GNU 版本的命令常常带额外选项，macOS 的 BSD 版本也有差异。写脚本时如果追求最强可移植性，最好使用 POSIX 标准里的通用写法，而不是某个发行版的专属扩展。

总结

POSIX 是 Unix 世界的一本“共同约定”。它规定了 Shell 命令、文件路径、进程管理、线程、信号和系统调用等接口，让不同系统之间保持足够的相似性。

Linux 和 macOS 的内核、血统完全不同，但你在它们终端里输入的命令却很像，正是因为它们都讲 POSIX 这门“普通话”。理解了 POSIX，你就能明白：操作系统之间的相似，不只是长得像，而是背后有一整套标准在支撑。
