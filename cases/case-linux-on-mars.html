Linux 在火星

第一章正文里讲过一个笑话：火星是宇宙中第二个 Linux 设备数超过 Windows 的行星。这话当然是调侃，但背后还真不是瞎说。NASA 的火星任务里，确实有一台运行 Linux 的小型计算机。

笑话背后的真实任务

2021 年 2 月，NASA 的毅力号（Perseverance）火星车降落在火星表面。和它一起去的，还有一架名叫 Ingenuity 的小型直升机。这架直升机是人类历史上第一架在另一个行星上进行动力飞行的航空器。

在地球上试飞直升机不难，但火星的环境完全不同：大气密度只有地球的 1% 左右，相当于地球上海拔三万米的高空。传统直升机靠足够稠密的空气产生升力，在火星上基本行不通。Ingenuity 必须通过极高的旋翼转速、轻量化的机身和精确的自主控制来克服这个难题。

Ingenuity 的个头很小，起飞重量只有大约 1.8 公斤，旋翼直径约 1.2 米，但它要完成的任务却很了不起：在火星稀薄的大气中自主起飞、悬停、导航和降落。为了完成这些任务，它配了一套高通骁龙 801 处理器作为飞行控制计算机。

一架运行 Linux 的直升机

Ingenuity 的飞行控制计算机运行的是 Linux 内核。这让它成为第一台在另一个行星上运行 Linux 的航空器，也是那个火星笑话真正的底气来源。

除了 Linux 内核，NASA 还在 Ingenuity 上用了 F´（读作 F Prime）框架。F´ 是 NASA 开源的飞行软件框架，专门面向小型航天器。它的设计目标是模块化、可移植、易于测试。NASA 把 F´ 的源码公开放在 GitHub 上，任何人都可以查看和学习。

航天器为什么选择 Linux

航天任务对软件的要求很苛刻：稳定、可控、可定制，出了问题还能自己排查。Linux 恰好能满足这些需求。

Linux 内核经过多年验证，稳定性有口皆碑。它已经在地球上无数服务器、嵌入式设备和科研仪器里跑了数十年，内核社区积累的大量测试和修复经验，可以直接用到航天器的开发流程里。

源码完全开放，NASA 的工程师就能按任务需求裁剪内核、去掉不需要的模块、精确控制每一处行为。深空任务的计算资源极其有限，每一兆内存和每一赫兹 CPU 都很珍贵。能自己决定内核里保留什么、去掉什么，这是商业闭源操作系统很难给的自由。

Linux 还不需要授权费用，对预算吃紧的科研项目来说是实实在在的优势。NASA 可以把有限的资金花在硬件、测试和人才培养上，而不是花在软件许可上。

庞大的开源社区也意味着遇到问题时，可参考的资源和经验非常丰富。从驱动适配到安全补丁，从实时性优化到交叉编译，地球上已经有大量工程师做过类似的工作。Ingenuity 的飞行软件框架 F´ 本身也是 NASA 开源出来的成果，这又反向回馈了开源社区。

不只在火星

Linux 在航天领域的应用并不止于 Ingenuity。国际空间站（ISS）上也曾迁移到基于 Linux 的计算机系统，用于处理实验控制和日常计算任务。地面上，大量的科学计算、天文观测、气象模拟和粒子物理实验设备也运行在 Linux 上。

再把视野放宽一点，SpaceX 的龙飞船和猎鹰火箭的飞行控制软件中也使用了 Linux 和大量开源工具；ESA、JAXA 等航天机构的不少任务同样依赖 Linux 生态。开源、可靠、可审计的软件，正在成为现代航天工程的常见选择。

火星笑话听起来轻松，但它背后是一个真实情况：Linux 已经不只是服务器机房里的操作系统，它真的飞出了地球。
