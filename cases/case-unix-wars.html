Unix 战争与 BSD 诉讼案

Unix 最早是贝尔实验室里几个工程师“为了自己用着方便”而做出来的系统，后来却成了学术界和黑客圈里大家共用的公共财富。可到了 1980 年代，这份财富忽然被贴上了价签，变成了一场漫长商业混战的导火索。

理解这段历史，能帮我们回答一个关键问题：为什么 Linux 会在 1991 年，以那样一种方式出现？说起来，那并不是 Unix 世界之外的偶然事件，而是 Unix 自己把门推开了一条缝。

这里要说的“Unix 战争”，核心就是 AT&T 和加州大学伯克利分校 BSD 之间围绕 Unix 版权与商业控制权的长期争夺。

1. 从开放到封闭

Unix 诞生于贝尔实验室，但 AT&T 早期根本没把它当主营业务。你可能会问：AT&T 不是电话公司吗？没错，当时它正因为反垄断案被美国政府盯着，禁止进入计算机软件销售市场。于是 Unix 的源码几乎以象征性的价格发给了大学，谁都能看、都能改。

加州大学伯克利分校就是其中最大的受益者之一。一帮学生和研究人员在源码上改来改去，慢慢搞出了 BSD（Berkeley Software Distribution）。那时候的 Unix，更像是实验室里的共享工具，而不是货架上的商品。

转折发生在 1984 年。美国政府依据反垄断法把 AT&T 的贝尔系统拆分了，禁令随之解除。AT&T 一看，原来手里还握着 Unix 这块宝贝，于是立刻把它变成真正的商品：商业版本卖得贵，源码也不再随便分发。Unix 从一个大家都能研究的开放系统，迅速变成了封闭的商业软件。

2. BSD 的崛起

AT&T 忙着把 Unix 变成摇钱树的同时，伯克利那帮人却在另一条路上越走越远。他们重写大量内核代码，把 TCP/IP 网络协议栈塞了进去，还发展出独立的 BSD 许可证，陆续推出 4.2BSD、4.3BSD 等版本。

到了 1980 年代末，BSD 已经成为学术机构和互联网基础设施里非常活跃的力量。很多人甚至觉得，BSD 有望成为 Unix 世界事实上的标准。但问题在于，BSD 最早毕竟是从 AT&T 的 Unix 代码里长出来的，版权归谁，始终是一笔糊涂账。

3. 诉讼爆发

1992 年，AT&T 的子公司 Unix System Laboratories（USL）把 BSDi 和加州大学伯克利分校告上了法庭，指控 BSD 侵犯 Unix 版权、泄露商业秘密，还滥用了商标。诉讼发起的时间点很微妙——个人电脑和互联网正好开始爆发，谁都想抢下这块新大陆。

这场官司一拖就是近两年。期间，BSD 的代码地位陷入了法律不确定性，厂商和研究机构都不敢再放心地基于 BSD 做新产品。原本势头很猛的 BSD，发展速度明显慢了下来。

4. 和解与余波

1994 年，双方终于达成和解。条件之一是 BSD 必须删掉约三处被认定来自 AT&T Unix 的核心文件，然后以 4.4BSD-Lite 的形式重新发布。官司在法律意义上结束了，但 BSD 错过的两年却补不回来。

之后，BSD 社区分化成 FreeBSD、OpenBSD、NetBSD 等几个分支，在服务器和网络设备领域一直有自己的一席之地。不过，这场诉讼确实让 BSD 错过了 1990 年代初期个人计算机和互联网基础设施高速扩张的最佳时机。

5. 为什么说这场诉讼给了 Linux 机会

1991 年，林纳斯·托瓦兹开始写 Linux 的时候，他并不是在真空里工作。他想要一个自由的、类 Unix 的内核，而当时 BSD 正被官司缠得脱不开身，Minix 又有教学上的限制。Linux 的出现，正好填上了这个空档。

还有一点很关键：Linux 从一开始就没有任何 AT&T 的代码，法律上是干净的。等到 1994 年 BSD 诉讼和解时，Linux 已经通过 GPL 协议聚集起全球开发者，发展势头已经形成。历史没法假设，但如果没有这场官司拖慢 BSD，Linux 是否还能后来居上，恐怕就是另一个故事了。
